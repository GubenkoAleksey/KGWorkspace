# План переходу на модульну Clean Architecture з MVI та Hilt

Цей план описує кроки для реструктуризації проекту `FirestoreApp` з моноліту в модульну структуру.

## Етап 1: Налаштування Hilt та базової інфраструктури
1.  **Додавання Hilt**:
    *   Оновити `libs.versions.toml` (версії hilt та плагіна).
    *   Додати hilt плагін у кореневий `build.gradle.kts`.
    *   Налаштувати hilt у `app/build.gradle.kts`.
    *   Створити клас `FirestoreApplication` з анотацією `@HiltAndroidApp`.
2.  **Створення модуля `:core`**:
    *   Створити Gradle-модуль для спільного коду.
    *   Реалізувати базові класи для MVI: `BaseViewModel`, `ViewState`, `ViewIntent`, `ViewSideEffect`.

## Етап 2: Виділення шарів Data та Domain
1.  **Створення модуля `:domain`**:
    *   Чистий Kotlin-модуль.
    *   Перенесення бізнес-моделей (Entity).
    *   Визначення інтерфейсів репозиторіїв.
    *   Створення UseCases (наприклад, `GetEmployeeStatusUseCase`).
2.  **Створення модуля `:data`**:
    *   Android-модуль.
    *   Перенесення Room (Dao, Database) та Firestore логіки.
    *   Реалізація інтерфейсів репозиторіїв.
    *   Налаштування Hilt-модуля для надання залежностей (Database, Api, Repositories).

## Етап 3: Модуляризація фіч (Feature-based)
1.  **Створення модуля `:feature:home`**:
    *   Перенесення `HomeMenuScreen` та пов'язаного UI.
    *   Реалізація `HomeViewModel` у стилі MVI.
    *   Налаштування навігації всередині модуля.
2.  **Створення модуля `:feature:status`**:
    *   Перенесення логіки відображення статусів.
    *   Реалізація MVI (State, Intent, Effect).

## Етап 4: Фінальна збірка в `:app`
1.  Очищення модуля `:app` від бізнес-логіки.
2.  Налаштування головного `NavHost`, який об'єднує екрани з feature-модулів.
3.  Забезпечення впровадження залежностей через Hilt між модулями.

## Примітки щодо MVI:
*   Кожна фіча повинна мати свій `Contract` клас, що містить `State`, `Intent` та `Effect`.
*   ViewModel повинна наслідуватись від `BaseViewModel` з `:core`.
*   Compose-екрани повинні бути "stateless", отримуючи стан та надсилаючи інтенти.
